# TraderLog — Protokół Testowania

> **Wersja:** 1.0 · **Data:** Czerwiec 2026  
> **Cel:** Ocena czy aplikacja faktycznie buduje procesy i nawyki tradingowe.  
> **Dla:** Testerów zewnętrznych + automatyczny agent Claude Code.

---

## Część A — Protokół dla Testerów Ludzkich

### Jak testować

1. Pobierz dostęp od admina (dominskipatryk@gmail.com)
2. Używaj aplikacji przez **minimum 5 kolejnych dni roboczych**
3. Po każdym dniu wypełnij **Kartę Dzienną** (sekcja A.3)
4. Po 5 dniach wypełnij **Raport Końcowy** (sekcja A.4)
5. Wyślij raport na adres admina

---

## Część B — Automatyczny Agent Claude Code

### Jak uruchomić

W terminalu Claude Code wpisz:

```
/review --fix
```

Lub uruchom poniższy prompt jako nową rozmowę z Claude Code:

---

### B.1 Prompt Agenta Automatycznego

```
Jesteś agentem testującym aplikację TraderLog (plik preview.html).
Twoim zadaniem jest ocena czy aplikacja faktycznie buduje nawyki 
tradingowe — nie tylko czy technicznie działa.

Przeanalizuj kod pod kątem poniższych obszarów i wygeneruj raport.

=== OBSZAR 1: MECHANIZMY NAWYKOWE ===
Sprawdź czy w kodzie istnieją i działają:
- Streak (passa) — czy jest zliczana, wyświetlana, resetowana prawidłowo?
- Habit Tracker — czy kolorowanie dni zależy od faktycznych akcji usera?
- Powiadomienia — czy są zaplanowane o sensownych porach?
- Dzienny Score 0-100 — jak jest liczony, czy motywuje?

Dla każdego: DZIAŁA / CZĘŚCIOWE / BRAK + krótkie uzasadnienie.

=== OBSZAR 2: PĘTLE RYTUAŁÓW ===
Sprawdź flow Rytuału Otwarcia i Zamknięcia:
- Czy rytuał jest wymagany/sugerowany przed dostępem do dashboardu?
- Czy dane z rytuału zamknięcia są zapisywane i widoczne?
- Czy jest logika "nie możesz handlować bez rytuału"?
- Czy jest możliwość pominięcia i jakie są konsekwencje?

=== OBSZAR 3: PODSUMOWANIA ===
Sprawdź logikę podsumowań tygodniowych:
- Czy istnieje rozróżnienie między "dane istnieją" a "user zatwierdził"?
- Czy kolumna reviewed/reviewed_at jest używana?
- Czy jest badge "Do uzupełnienia" vs "Zamknięty"?
- Czy w widoku weekly review są transakcje z tego tygodnia?

=== OBSZAR 4: ONBOARDING ===
Sprawdź czy nowy użytkownik jest prowadzony przez setup:
- Czy jest ekran powitalny / onboarding flow?
- Czy jest sugestia dodania pierwszego konta?
- Czy są podpowiedzi co zrobić jako pierwsze?

=== OBSZAR 5: FEEDBACK LOOP ===
Sprawdź czy aplikacja zamyka pętlę:
- Ankieta post-trade — czy dane z ankiety są używane w statystykach?
- Profil psychologiczny — czy bazuje na realnych danych transakcyjnych?
- Czy jest jakikolwiek mechanizm "wyciągania wniosków" z danych usera?

=== OUTPUT ===
Wygeneruj raport w formacie:

## Raport Agenta — [data]

### Ocena ogólna: X/10

### Obszar 1: Mechanizmy nawykowe — X/10
[szczegóły]

[szczegóły]

### Obszar 3: Podsumowania — X/10
[szczegóły]

### Obszar 4: Onboarding — X/10
[szczegóły]

### Obszar 5: Feedback loop — X/10
[szczegóły]

### Top 3 krytyczne luki:
1.
2.
3.

### Top 3 mocne strony:
1.
2.
3.

### Rekomendacje (kolejność priorytetów):
1.
2.
3.
```

---

### B.2 Interpretacja Wyników Agenta

| Wynik | Interpretacja | Akcja |
|---|---|---|
| 8–10/10 | Aplikacja buduje nawyki | Gotowa do szerszych testów |
| 6–7/10 | Mechanizmy istnieją, ale z lukami | Zamknąć luki przed skalowaniem |
| 4–5/10 | Podstawy są, brak spójności | Bugfixes + redesign flow |
| < 4/10 | Aplikacja nie buduje nawyku | Priorytet: fundamentalny redesign |
